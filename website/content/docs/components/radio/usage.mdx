## Import

```js
import { RadioGroup } from '@chakra-ui/react'
```

### Usage

```jsx
function RadioExample() {
  const [value, setValue] = React.useState('1')
  return (
    <RadioGroup.Root onChange={setValue} value={value}>
      <Stack direction='row'>
        <RadioGroup.Item value='1'>
          <RadioGroup.ItemControl />
          <RadioGroup.ItemText>First</RadioGroup.ItemText>
        </RadioGroup.Item>
        <RadioGroup.Item value='2'>
          <RadioGroup.ItemControl />
          <RadioGroup.ItemText>Second</RadioGroup.ItemText>
        </RadioGroup.Item>
        <RadioGroup.Item value='3'>
          <RadioGroup.ItemControl />
          <RadioGroup.ItemText>Third</RadioGroup.ItemText>
        </RadioGroup.Item>
      </Stack>
    </RadioGroup.Root>
  )
}
```

### Radio with custom color

You can override the `colorScheme` of the `RadioGroup.Item` to any color key
specified in `theme.colors`.

```jsx
<RadioGroup.Root defaultValue='2'>
  <Stack spacing={5} direction='row'>
    <RadioGroup.Item colorScheme='red' value='1'>
      <RadioGroup.ItemControl />
      <RadioGroup.ItemText>Radio</RadioGroup.ItemText>
    </RadioGroup.Item>
    <RadioGroup.Item colorScheme='green' value='2'>
      <RadioGroup.ItemControl />
      <RadioGroup.ItemText>Radio</RadioGroup.ItemText>
    </RadioGroup.Item>
  </Stack>
</RadioGroup.Root>
```

### Radio sizes

The checkbox comes with 3 sizes.

```jsx
<Stack>
  <RadioGroup.Item size='sm' name='1' colorScheme='red'>
    <RadioGroup.ItemControl />
    <RadioGroup.ItemText>Radio</RadioGroup.ItemText>
  </RadioGroup.Item>
  <RadioGroup.Item size='md' name='1' colorScheme='green'>
    <RadioGroup.ItemControl />
    <RadioGroup.ItemText>Radio</RadioGroup.ItemText>
  </RadioGroup.Item>
  <RadioGroup.Item size='lg' name='1' colorScheme='orange' defaultChecked>
    <RadioGroup.ItemControl />
    <RadioGroup.ItemText>Radio</RadioGroup.ItemText>
  </RadioGroup.Item>
</Stack>
```

### Custom Radio Buttons

In some cases, you might need to create components that work like radios but
don't look like radios. Chakra exports `useRadio`, and `useRadioGroup` hooks to
help with this scenario. Here's what you need to do:

1. Create a component that consumes the `useRadio` hook.
2. Use the `useRadioGroup` hook to control a group of custom radios.

You can head on over to the pages for the
[useRadio](https://chakra-ui.com/docs/styled-system/component-hooks/use-radio)
and
[useRadioGroup](https://chakra-ui.com/docs/styled-system/component-hooks/use-radio-group)
hooks to see more detail about their uses.

> Please be aware that the example below should only be used if you really need
> a radio button for data collection purposes. If you want to toggle between
> different content on activation of a button use the `Tabs` component.

```jsx manual=true
// 1. Create a component that consumes the `useRadio` hook
function RadioCard(props) {
  const { getInputProps, getRadioProps } = useRadio(props)

  const input = getInputProps()
  const checkbox = getRadioProps()

  return (
    <Box as='label'>
      <input {...input} />
      <Box
        {...checkbox}
        cursor='pointer'
        borderWidth='1px'
        borderRadius='md'
        boxShadow='md'
        _checked={{
          bg: 'teal.600',
          color: 'white',
          borderColor: 'teal.600',
        }}
        _focus={{
          boxShadow: 'outline',
        }}
        px={5}
        py={3}
      >
        {props.children}
      </Box>
    </Box>
  )
}

// Step 2: Use the `useRadioGroup` hook to control a group of custom radios.
function Example() {
  const options = ['react', 'vue', 'svelte']

  const { getRootProps, getRadioProps } = useRadioGroup({
    name: 'framework',
    defaultValue: 'react',
    onChange: console.log,
  })

  const group = getRootProps()

  return (
    <HStack {...group}>
      {options.map((value) => {
        const radio = getRadioProps({ value })
        return (
          <RadioCard key={value} {...radio}>
            {value}
          </RadioCard>
        )
      })}
    </HStack>
  )
}

render(<Example />)
```

### Note about `name` prop

We recommend passing the `name` prop to the `RadioGroup.Root` component, instead
of passing it to each `RadioGroup.Item` component. By default, the `name` prop
of the RadioGroup takes precedence.

```jsx live=false
// Do this ✅
<RadioGroup.Root name="form-name">
  <RadioGroup.Item>Radio 1</RadioGroup.Item>
  <RadioGroup.Item>Radio 2</RadioGroup.Item>
</RadioGroup.Root>

// Don't do this ❌
<RadioGroup.Root >
  <RadioGroup.Item name="form-name">Radio 1</RadioGroup.Item>
  <RadioGroup.Item name="form-name">Radio 2</RadioGroup.Item>
</RadioGroup.Root>
```
