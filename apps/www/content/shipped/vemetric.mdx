## 👋🏽 A bit of context

[Vemetric](https://vemetric.com/) is a privacy-first, open-source analytics
platform that helps you understand where your users are coming from, and how you
make them stay.

I'm Dominik Sumer, an indie founder building Vemetric.

I've been using Chakra for more than 5 years now. The way it lets me design my
components, leveraging my existing CSS knowledge plus having type safety and an
easy way to write responsive styles, is still unmatched.

## What tripped me up

In the past there were problems like making Chakra work with Framer Motion or
other libraries. But it became so mature with version 3 that I think it fits
very well into almost any modern frontend codebase.

## Advice for developers

Use Chakra if you want to build a design system and strong foundation for your
application's frontend.

<QuoteCard
  quote="Use Chakra if you want to build a design system and strong foundation for your application's frontend."
  image="/shipped/dominik-sumer.png"
/>

I have to be a bit honest in that I don't know how Chakra is going to develop in
the direction of dropping emotion and eliminating runtime CSS-in-JS, which is
still a bit of a bummer for some projects, also for me when I want to build
highly performant landing pages, where I use PandaCSS instead. That's where I'm
personally the most interested in how Chakra is going to develop in the future.
