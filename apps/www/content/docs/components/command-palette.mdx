## Usage

```jsx
import { CommandPalette } from "@chakra-ui/react"
```

```jsx
<CommandPalette.Root>
  <CommandPalette.Label />
  <CommandPalette.Control>
    <CommandPalette.Indicator />
    <CommandPalette.Input />
  </CommandPalette.Control>
  <CommandPalette.List>
    <CommandPalette.ItemGroup>
      <CommandPalette.ItemGroupLabel />
      <CommandPalette.Item>
        <CommandPalette.ItemText />
        <CommandPalette.ItemCommand />
        <CommandPalette.ItemIndicator />
      </CommandPalette.Item>
    </CommandPalette.ItemGroup>
    <CommandPalette.Empty />
  </CommandPalette.List>
  <CommandPalette.Footer />
</CommandPalette.Root>
```

To setup the command palette, use `useListCollection` to manage the
[list collection](https://ark-ui.com/docs/collections/list-collection), and
`useFilter` to filter the items as the user types.

By default the palette runs commands: use `onSelect` to execute them, and no
selected state persists afterwards. To build a picker where selection persists,
set `selectionMode` to `single` or `multiple`.

The command palette renders inline. For the classic `⌘K` overlay experience,
compose it with `Dialog` as shown in the [Within Dialog](#within-dialog)
example.

> When controlling the palette with `RootProvider` and the `useListbox` hook,
> configure selection on the hook — the `selectionMode="none"` default only
> applies to `Root`.

### Coming from cmdk

| cmdk                            | Chakra UI                                                |
| ------------------------------- | -------------------------------------------------------- |
| `<Command>`                     | `<CommandPalette.Root collection={collection}>`          |
| `<Command.Input>`               | `<CommandPalette.Input>` + `useFilter`                   |
| `<Command.List>`                | `<CommandPalette.List>`                                  |
| `<Command.Item onSelect={fn}>`  | `<CommandPalette.Item item={item}>` + `onSelect` on Root |
| `<Command.Group heading="...">` | `<CommandPalette.ItemGroup>` + `ItemGroupLabel`          |
| `<Command.Empty>`               | `<CommandPalette.Empty>`                                 |
| `<Command.Dialog>`              | Compose with `Dialog` (see [below](#within-dialog))      |

The key difference: `cmdk` owns filtering and ranking internally, while the
command palette treats data as userland. You own the collection and how it
shrinks — swap in fuzzy matching, server search, or debouncing without fighting
a built-in filter.
