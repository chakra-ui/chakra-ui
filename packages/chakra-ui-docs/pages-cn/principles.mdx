# 设计原则

查克拉UI遵从组件一致的原则。理解以下概念有助于你为查克拉UI做出更大的贡献。

我们的目标是设计简易、模块化的组件，以解决现实中的设计难题。
因此，我们提出如下原则，让我们不忘初心。

- **样式属性：** 所有组件样式可被覆写或通过样式属性扩展，
  以减少`css`属性或`styled()`的使用。
  建议用`Box`或`PseudoBox`组合新组件。

- **简易：** 尽量保持组件接口简单易懂，
  同时展示现实场景中的使用。

- **模块化：** 为了降低复杂度、方便组装，
  将组件分拆成具有最少属性的部件。
  这样能确保样式和功能灵活、可延展。

- **无障碍：** 创建组件时，考虑无障碍使用性。
  包括键盘导航、聚焦管理、颜色对比、旁白和正确的`aria-*`属性。

- **深色模式：** 尝试让组件兼容深色模式。
  使用钩子`ColorModeProvider`和`useColorMode`处理样式。
  [深色模式了解更多](/color-mode)

- **属性命名：** 众所周知，命名是最让人头痛的。
  通常名称应该揭示他的功能。
  布尔属性应该用`does`、`has`、`is`、`should`等助动词命名。
  例如`Button`的属性`isDisabled`、`isLoading`等等。
